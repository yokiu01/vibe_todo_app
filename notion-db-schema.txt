```
Goals (목표)
  ↓ relation
Projects (프로젝트)
  ↓ relation
Tasks (할 일)

Areas · Resources (영역 · 자원)
  ↔ relation with all above
  ↔ relation with Notes

Notes (노트)
  ↔ relation with Tasks, Projects, Areas
  ↔ self-relation with other Notes
```

## API 사용 시 주요 참고사항

### Date 속성 사용법
날짜 속성은 3개의 확장 필드로 분리됩니다:
- `date:{속성명}:start` - 시작 날짜 (ISO-8601 형식)
- `date:{속성명}:end` - 종료 날짜 (선택사항, 범위인 경우만)
- `date:{속성명}:is_datetime` - 날짜/시간 여부 (0 또는 1)

### Checkbox 속성 사용법
- `"__YES__"` = true (체크됨)
- `"__NO__"` = false (체크 안됨)
- `NULL` = 기본값 false

### Relation 속성 사용법
- JSON 배열 형식의 페이지 URL 목록
- 예: `["https://notion.so/page-id-1", "https://notion.so/page-id-2"]`

### Multi-select 속성 사용법
- JSON 배열 형식의 옵션 이름 목록
- 예: `["창의성", "컴퓨터", "집에서"]`

### Formula와 Rollup 속성
- 읽기 전용 (read-only)
- API를 통해 직접 수정 불가
- 관련된 속성값이 변경되면 자동으로 재계산됨

## 워크스페이스 구조

이 데이터베이스들은 **PARA** (Projects, Areas, Resources, Archives) 시스템을 기반으로 한 **Second Brain OS** 구조를 따릅니다.

### 계층 구조
```
PARA
└── Menu
    └── Databases
        ├── Tasks Database
        ├── Projects Database
        ├── Goals Database
        ├── Areas · Resources Database
        └── Notes Database
```

### GTD (Getting Things Done) 워크플로우
1. **수집 (Collection)**: Tasks 또는 Notes에 아이템 추가
2. **명료화 (Clarification)**: Tasks의 "명료화" 속성으로 분류
3. **조직화 (Organization)**: Projects, Areas, Goals와 연결
4. **검토 (Review)**: 상태 및 진행률 확인
5. **실행 (Engagement)**: "다음 행동 상황"에 따라 실행
